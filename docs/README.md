# Documentation map

This repository now keeps a sorted documentation map by purpose.

## Preferred documentation layout

- `docs/guides/` — shortest practical paths for normal users
- `docs/reference/` — command and runtime reference
- `docs/platform/` — platform-specific launcher guidance
- `docs/roadmap/` — forward-looking product and version plans
- `docs/release/` — maintainer release checks

## Compatibility note

The older top-level or flat `docs/` files are still kept for compatibility and existing links.

The sorted folders are the preferred reading order going forward.
